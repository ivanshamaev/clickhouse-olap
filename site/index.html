Why ClickHouse OLAP

Everything Excel pivot tables lacked

Built for analysts who outgrew Excel's million-row limit but don't want to learn a new BI tool.

⚡

Sub-second pivots

ClickHouse aggregates hundreds of millions of rows in milliseconds. The Rust engine adds < 5 ms overhead.

🔒

Zero raw data exposure

Only pre-aggregated results leave ClickHouse. Row-level data never reaches the browser or add-in.

📊

Native Excel UX

Drag-and-drop dimensions and measures in a familiar task pane. Results land directly in your workbook.

🦀

Stateless Rust engine

No session state between requests — scale horizontally behind any load balancer, zero memory leaks.

Architecture

Thin client, heavy engine

Excel Office.js add-in
→
Rust Engine /v1/query · HTTPS
→
ClickHouse SQL · JSONEachRow
  • 🎯 Add-in serialises the user's pivot intent (rows, columns, measures, filters) into a JSON request and renders the returned compact result grid.
  • 🦀 Rust engine validates the request against the semantic model, generates parameterised aggregating SQL, executes one sub-query per totals level, shapes the result into a pivot matrix and caches it.
  • 🗄 ClickHouse does all the heavy lifting — GROUP BY, ORDER BY, aggregation functions. Raw rows never leave it.
Quick Start

Up and running in 3 steps

1

Deploy the Rust engine on your server

Install ClickHouse, build the engine from source (or download a release binary), configure config.toml with your ClickHouse credentials and semantic model, and start the systemd service.

Engine deployment guide →
2

Put the engine behind HTTPS

Office.js requires HTTPS even for local connections. Use nginx + Let's Encrypt on your domain. The nginx configuration takes about 5 minutes.

3

Install the Excel add-in

Download manifest.xml and sideload it in Excel. Enter your engine URL in the add-in settings, click Test Connection, and start building pivot tables.
