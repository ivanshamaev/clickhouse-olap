Excel Add-in Setup

Download manifest.xml, sideload the add-in into Excel on Windows or Mac, and connect it to your deployed engine.

Prerequisites

  • Microsoft Excel desktop — Windows 2016 or later, macOS 2019 or later (Excel on the web also works)
  • The Rust engine is deployed and accessible via HTTPS (e.g. https://engine.yourdomain.com)
  • A stable internet connection to download manifest.xml
HTTPS is required Office.js add-ins must be served over HTTPS — even for localhost connections. If your engine is not yet behind HTTPS, follow the nginx + HTTPS section first.

Download manifest.xml

Download the add-in manifest from the latest GitHub release. The manifest tells Excel where to load the add-in UI from (GitHub Pages) and which ribbon buttons to add.

⬇ Download manifest.xml

Or download from the command line:

curl -L -o manifest.xml \
  https://github.com/ivanshamaev/clickhouse-olap/releases/latest/download/manifest.xml

Install on Windows

1

Open Excel

Open any workbook in Microsoft Excel for Windows (or create a new blank workbook).

2

Navigate to Add-ins

Click Insert tab → Add-ins → My Add-ins.

3

Upload the manifest

In the My Add-ins dialog, click Upload My Add-in (bottom-left link). Browse to the downloaded manifest.xml file and click Upload.

4

Open the add-in

A ClickHouse OLAP button appears on the Home tab. Click it to open the task pane.

Tip — network shares for teams For organisations with multiple Excel users, put manifest.xml on a shared network folder and have each user point the Trusted Add-in Catalog to that share. Updates then propagate automatically.

Install on macOS

1

Locate the WEF folder

Create (if missing) the folder:

~/Library/Containers/com.microsoft.Excel/Data/Documents/wef/
2

Copy manifest.xml

Copy the downloaded manifest.xml into the wef/ folder.

3

Restart Excel and open the add-in

Restart Excel. Go to Insert → Add-ins → My Add-ins. The ClickHouse OLAP add-in will appear in the Developer Add-ins section.

Connect to engine

1

Open Settings

In the add-in task pane click the ⚙ Settings icon (top-right).

2

Enter Engine URL

In the Engine URL field type your engine's base URL, e.g.:

https://engine.yourdomain.com

No trailing slash, no /v1 path — the add-in appends that automatically.

3

Test Connection

Click Test Connection. A green ✓ Connected message confirms the engine is reachable and the API version is compatible.

4

Save settings

Click Save. The URL is persisted in the workbook so you don't need to re-enter it.

Build your first pivot table

1

Select a model

The add-in lists all semantic models available on the engine. Select the model that contains your data (e.g. Sales).

2

Drag dimensions and measures

Drag available fields into the Rows, Columns, and Measures areas. For date fields, select the granularity (Year / Quarter / Month / Day).

3

Apply filters (optional)

Click + Filter to restrict the data by region, date range, or any other dimension.

4

Click Run

Click ▶ Run. The pivot result is written directly into the selected cell range in your workbook. Subtotals and grand totals are included if toggled on.

Troubleshooting

SymptomCause & fix
Add-in task pane shows blank white page The add-in assets could not be loaded. Check that GitHub Pages is reachable (ivanshamaev.github.io/clickhouse-olap) and your Excel has internet access.
"Connection failed" on Test Connection Engine URL is wrong, the engine is down, or HTTPS certificate is invalid. Run curl https://engine.yourdomain.com/v1/health from a terminal to diagnose.
"CORS error" in browser dev tools nginx CORS headers don't include the add-in origin. Update the Access-Control-Allow-Origin header in nginx to match the GitHub Pages URL.
"MODEL_NOT_FOUND" The selected model does not exist on the engine. Check /etc/clickhouse-olap/models/ and restart the engine service.
"PREAGGREGATE_TOO_LARGE" The pivot dimensions would produce too many groups. Add a filter, reduce dimensions, or increase max_groups in config.toml.
Old version of add-in loads Clear the Office add-in cache. In Excel: File → Options → Trust Center → Trust Center Settings → Trusted Add-in Catalogs → Clear cache, then restart Excel.

Update the add-in

The add-in UI is hosted on GitHub Pages and updates automatically when a new version is deployed — no action needed on your side for UI changes.

If a new manifest.xml is released (which happens only on breaking API changes), re-download and re-sideload it using the same steps as the initial installation. Watch the GitHub Releases page to know when a manifest update is required.

All set! You now have a fully functional ClickHouse OLAP pivot table add-in in Excel. If you have questions or find a bug, open an issue on GitHub.
