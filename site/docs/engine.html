Engine Deployment

Build the stateless Rust engine, configure semantic models, set up systemd + nginx + Let's Encrypt, and verify the deployment end-to-end.

Prerequisite Complete the ClickHouse Setup guide first. The engine needs olap_user credentials and at least one table to query.

Install Rust

The engine requires the stable Rust toolchain. Install via rustup:

# As the user that will run the engine (e.g. "engine")
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
source ~/.cargo/env

# Verify
rustc --version   # should be stable 1.80+
cargo --version

Optionally create a dedicated system user first:

useradd -m -s /bin/bash engine

Build the engine

Option A — build from source (recommended)

# Clone the repository
git clone https://github.com/ivanshamaev/clickhouse-olap.git
cd clickhouse-olap/engine

# Build the optimised release binary (~3–5 minutes on first build)
cargo build --release

# Binary is at:
ls -lh target/release/engine

Option B — download a release binary

VERSION="0.1.0"
curl -L "https://github.com/ivanshamaev/clickhouse-olap/releases/download/v${VERSION}/clickhouse-olap-engine-v${VERSION}-x86_64-unknown-linux-gnu.tar.gz" \
  | tar -xz

sudo mv engine /usr/local/bin/clickhouse-olap-engine
sudo chmod +x /usr/local/bin/clickhouse-olap-engine

Configuration

Create the config directory and copy the example config:

sudo mkdir -p /etc/clickhouse-olap/models
sudo chown -R engine:engine /etc/clickhouse-olap

Create /etc/clickhouse-olap/config.toml:

[server]
# Engine listens locally; nginx proxies external HTTPS traffic
bind = "127.0.0.1:3000"

[clickhouse]
url      = "http://127.0.0.1:8123"
database = "olap"
username = "olap_user"
password = "REPLACE_WITH_STRONG_PASSWORD"
query_timeout_secs    = 30
max_preaggregate_rows = 500000

[models]
path = "/etc/clickhouse-olap/models"

[cache]
capacity = 1000
ttl_secs = 300

[limits]
max_cells  = 500000
max_groups = 200000
Security Set file permissions: chmod 640 /etc/clickhouse-olap/config.toml. The password is stored in plain text — restrict read access to the engine user only.

Semantic models

Each .toml file in the models/ directory defines one semantic model (one pivot-able dataset). Copy the example model from the repository: 

cp /home/engine/clickhouse-olap/engine/config/models/sales_model.toml \
   /etc/clickhouse-olap/models/

Or create a model manually. Example sales_model.toml:

[model]
id    = "sales_model"
label = "Sales"
table = "olap.sales"

[[dimensions]]
field = "region"
label = "Region"
type  = "string"

[[dimensions]]
field = "country"
label = "Country"
type  = "string"

[[dimensions]]
field = "product_category"
label = "Product Category"
type  = "string"

[[dimensions]]
field      = "order_date"
label      = "Order Date"
type       = "date"
hierarchy  = ["year", "quarter", "month"]

[[measures]]
id          = "revenue"
field       = "amount"
label       = "Revenue"
allowed_agg = ["sum", "avg", "min", "max"]
format      = "currency"

[[measures]]
id          = "orders"
field       = "order_id"
label       = "Orders"
allowed_agg = ["count_distinct"]

[[measures]]
id          = "quantity"
field       = "qty"
label       = "Quantity"
allowed_agg = ["sum", "avg"]

[guards]
max_groups             = 200000
high_cardinality_fields = ["order_id", "customer_id"]
High-cardinality guard Fields listed in high_cardinality_fields are blocked from GROUP BY unless the request includes a top_n limit or an equality/in filter on that field. This prevents accidental multi-million-group queries from reaching ClickHouse.

systemd service

sudo tee /etc/systemd/system/clickhouse-olap-engine.service > /dev/null << 'EOF'
[Unit]
Description=ClickHouse OLAP Engine
After=network.target clickhouse-server.service
Requires=clickhouse-server.service

[Service]
Type=simple
User=engine
Group=engine
WorkingDirectory=/home/engine
ExecStart=/home/engine/clickhouse-olap/engine/target/release/engine \
    --config /etc/clickhouse-olap/config.toml
Restart=on-failure
RestartSec=5s
StandardOutput=journal
StandardError=journal
SyslogIdentifier=clickhouse-olap-engine

# Security hardening
NoNewPrivileges=yes
ProtectSystem=strict
ProtectHome=read-only
ReadWritePaths=/tmp

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now clickhouse-olap-engine
sudo systemctl status clickhouse-olap-engine

Check the logs:

journalctl -u clickhouse-olap-engine -f

nginx + HTTPS

Office.js requires HTTPS even for local connections. You need a real domain pointed at your server's IP. 

# Install nginx and certbot
sudo apt install -y nginx certbot python3-certbot-nginx

# Obtain an SSL certificate (domain must already resolve to this IP)
sudo certbot --nginx -d engine.yourdomain.com

# Create nginx site config
sudo tee /etc/nginx/sites-available/clickhouse-olap << 'EOF'
server {
    listen 80;
    server_name engine.yourdomain.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name engine.yourdomain.com;

    ssl_certificate     /etc/letsencrypt/live/engine.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/engine.yourdomain.com/privkey.pem;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         HIGH:!aNULL:!MD5;

    # CORS — restrict to your add-in origin
    add_header Access-Control-Allow-Origin  "https://ivanshamaev.github.io" always;
    add_header Access-Control-Allow-Methods "GET, POST, OPTIONS"             always;
    add_header Access-Control-Allow-Headers "Content-Type"                   always;

    if ($request_method = OPTIONS) {
        return 204;
    }

    location /v1/ {
        proxy_pass         http://127.0.0.1:3000;
        proxy_set_header   Host              $host;
        proxy_set_header   X-Real-IP         $remote_addr;
        proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
        proxy_send_timeout 60s;
    }
}
EOF

sudo ln -sf /etc/nginx/sites-available/clickhouse-olap \
            /etc/nginx/sites-enabled/clickhouse-olap
sudo nginx -t && sudo systemctl reload nginx
Let's Encrypt auto-renewal Certbot installs a systemd timer (certbot.timer) that automatically renews certificates before they expire. Verify it's active: systemctl status certbot.timer

Firewall

sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP → redirect to HTTPS
sudo ufw allow 443/tcp   # HTTPS (nginx → engine)
sudo ufw enable
sudo ufw status

# Do NOT expose these ports:
# 8123 — ClickHouse HTTP (127.0.0.1 only)
# 3000 — engine HTTP    (127.0.0.1 only)

Verify

From the server (loopback)

# Health check
curl http://127.0.0.1:3000/v1/health
# → {"status":"ok"}

# List loaded models
curl http://127.0.0.1:3000/v1/models
# → [{"model_id":"sales_model","label":"Sales"}]

# Run a test query
curl -s -X POST http://127.0.0.1:3000/v1/query \
  -H "Content-Type: application/json" \
  -d '{
    "api_version":"1",
    "request_id":"test-001",
    "model_id":"sales_model",
    "rows":[{"field":"region"}],
    "columns":[],
    "measures":[{"id":"revenue","agg":"sum"}],
    "filters":[],
    "totals":{"row_subtotals":false,"column_subtotals":false,"grand_total":true},
    "limits":{}
  }' | python3 -m json.tool

From outside (via nginx + HTTPS)

curl https://engine.yourdomain.com/v1/health
# → {"status":"ok"}

Update engine

su - engine
cd clickhouse-olap
git pull origin main
cd engine
cargo build --release

sudo systemctl restart clickhouse-olap-engine
sudo systemctl status clickhouse-olap-engine

Logs & monitoring

# Engine logs (live)
journalctl -u clickhouse-olap-engine -f

# nginx access log
tail -f /var/log/nginx/access.log

# nginx error log
tail -f /var/log/nginx/error.log

# ClickHouse log
tail -f /var/log/clickhouse-server/clickhouse-server.log

# Resource usage
systemctl status clickhouse-olap-engine clickhouse-server

Security checklist

  • ClickHouse listens only on 127.0.0.1:8123
  • Engine listens only on 127.0.0.1:3000
  • UFW enabled: only ports 22, 80, 443 open
  • SSL certificate (Let's Encrypt) installed and auto-renewing
  • Dedicated ClickHouse user with GRANT ON olap.* only
  • engine user has no sudo privileges in production
  • NoNewPrivileges=yes and ProtectSystem=strict in systemd unit
  • CORS in nginx restricted to specific add-in origin (not *)
  • config.toml readable only by the engine user (chmod 640)
